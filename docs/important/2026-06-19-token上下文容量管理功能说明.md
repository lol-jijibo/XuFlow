# Token 维度上下文容量管理 — 功能说明

> 日期：2026-06-19  
> 分支：main  
> 涉及模块：Rust Core、Tauri Bridge、Vue Frontend

---

## 一、功能概述

为 Xuflow 实现了端到端的 **token 维度上下文容量管理**系统，包含三个核心能力：

1. **Token 估算** — 在每次 API 调用前后估算当前对话占用的 token 数量
2. **动态截断** — 当上下文接近模型窗口上限时，自动丢弃较早的对话轮次以释放空间
3. **可视化反馈** — 前端实时展示上下文用量（圆环进度、状态栏、悬停详情），截断时给出温和提示

此前 `AgentLoop.messages` 向量无限制增长，没有任何容量感知。前端上下文圆环是硬编码占位符（`12000/128000`），从未连接到实际数据。本次改动彻底解决了这一问题。

## 三、Token 估算算法

### 3.1 核心思路

不依赖 `tiktoken-rs`（编译笨重、需要模型词表文件），采用**字符级启发式估算**，结合 API 返回的实际 token 数进行校准。

### 3.2 字符分类

| 字符类型 | Unicode 范围 | 系数 | 说明 |
|----------|-------------|------|------|
| CJK 字符 | 中日韩统一表意文字、平假名、片假名、谚文 | **1.3** tokens/字符 | 这些字符在多数 subword tokenizer 中约 1~2 tokens |
| 非 CJK | 拉丁字母、空格、标点、数字 | **0.25** tokens/字符 | 英语约 4 字符/token |
| 结构化内容 | JSON/工具返回内容（符号密集） | **0.5** tokens/字符 | 偏代码密度 |

### 3.3 检测逻辑

- **CJK 判定**：检查字符是否落在 `\u{4E00}-\u{9FFF}`、`\u{3400}-\u{4DBF}`、`\u{F900}-\u{FAFF}`、`\u{3040}-\u{309F}`、`\u{30A0}-\u{30FF}`、`\u{AC00}-\u{D7AF}`
- **结构化判定**：若内容字符串以 `{` 或 `[` 开头且可解析为合法 JSON，或字符中字母数字占比 < 30%，则判定为结构化内容

### 3.4 双阶段校准

每次 API 调用会发送两次 `TokenUsage` 事件：

1. **`phase: "before"`** — 仅携带 `estimated`（启发式估算值）
2. **`phase: "after"`** — 携带 `estimated` + `actual`（API 返回的 `usage.total_tokens`），前端使用 `max(estimated, actual)` 作为显示值

### 3.5 可配置性

- 三个系数（CJK / 非CJK / 结构化）在设置页可按模型独立覆盖
- 默认系数对所有当前模型（DeepSeek、豆包、GLM）适用
- 高级用户可微调以提升特定模型的估算精度

## 四、动态截断策略

### 4.1 触发条件

- 当前估算 token 数 ≥ 上下文窗口的 **80%**

### 4.2 截断算法

```
1. 保护系统消息（第一条 system 角色消息永远保留）
2. 将剩余消息组织为"轮次"（turn）：
   一轮 = 一条 user 消息 + 后续所有 assistant/tool 消息（直到下一条 user 为止）
3. 从后往前标记最后 N 个轮次为"保护轮次"（N = min_user_turns，默认 3）
4. 从最旧的未保护轮次开始逐个丢弃
5. 每丢弃一轮重新估算，直到：
   - 用量降至 60% 以下，或
   - 只剩保护轮次
6. 发送 ContextTrimmed 事件通知前端
```

### 4.3 关键特性

- **原子性**：一轮工具调用（user → assistant(tool_calls) → tool results）要么全保留、要么全丢弃，不会中途腰斩
- **静默处理**：不向对话消息列表插入可见的系统消息，完全由前端 UI 层展示截断通知
- **触发时机**：在 `run()` 方法的两个位置触发：
  - 用户消息推入后、首次 API 调用前
  - 每轮工具返回后、下一轮 API 调用前（消息增长时）

### 4.4 与旧方案对比

| 维度 | 旧方案（计划 v1） | 新方案（计划 v2） |
|------|------------------|-------------------|
| 保留策略 | 固定保留最近 3 轮 | 基于 token 量动态保留，目标降至 60% |
| 轮数配置 | 硬编码不可配 | `min_user_turns` 可配（1-10） |
| 截断通知 | 插入系统消息到对话 | 静默处理，前端 UI 层短暂标签 |
| 原子性 | 无保证 | 整轮保留或整轮丢弃 |

## 五、StreamEvent 新增变体

### 5.1 TokenUsage

```rust
TokenUsage {
    phase: String,           // "before" | "after"
    estimated: u32,          // 启发式估算值（始终存在）
    actual: Option<u32>,     // API 返回的实际值（仅 phase="after"）
    context_window: u32,     // 当前模型上下文窗口大小
    context_remaining: u32,  // 剩余容量 = context_window - max(estimated, actual)
}
```

**发送时机**：
- 用户消息入队后（`phase: "before"`）
- 每轮 API 返回后（`phase: "after"`，携带 actual）
- 每轮工具处理完后、下一轮循环前（`phase: "before"`）

### 5.2 ContextTrimmed

```rust
ContextTrimmed {
    rounds_removed: u32,       // 丢弃了几轮对话
    tokens_freed: u32,          // 释放了多少 token
    current_usage_percent: u32, // 截断后的容量占比 (0-100)
    context_window: u32,
}
```

**发送时机**：仅在 `AgentLoop::trim_context()` 实际执行截断时发送一次。

### 5.3 Done（版本化）

```json
{ "v": 1, "usage": { "prompt_tokens": 12345, "completion_tokens": 678, "total_tokens": 13023 } }
```

- 前端检查 `v` 字段：有则解析 usage，无则回退到旧行为（忽略空 payload）
- 未来扩展只需递增版本号

## 六、前端可视化设计

### 6.1 上下文圆环（ChatPanel 页脚）

- **空态**：浅灰色圆环轮廓（`opacity: 0.35`）
- **填充**：SVG `<circle>` 前景弧线，`stroke-dasharray` = 百分比对应弧长（周长 39.27）
- **颜色告警**：

| 用量 | 颜色 | 色值 |
|------|------|------|
| < 50% | 绿色 | `#22c55e` |
| 50-80% | 黄色 | `#eab308` |
| 80-90% | 橙色 | `#f59e0b` |
| > 90% | 红色 | `#ef4444` |

- **工具提示**（悬停）：显示 `已用 X% · 剩余 Y tokens` + 详细 `估算值 / 窗口大小`

### 6.2 截断提示标签

- 位置：上下文圆环上方
- 内容："对话已自动整理"
- 行为：从 `opacity: 0` 淡入，2.5 秒后淡出（CSS animation）
- `pointer-events: none`，不干扰交互

### 6.3 StatusBar Token 标签

- 格式：`12K / 128K · 就绪 · v1.0.2`
- 颜色：跟随 `tokenWarningLevel`
- 字体：等宽 `SF Mono`

## 七、设置页配置项

路径：设置 → 上下文管理

| 配置项 | 类型 | 范围 | 默认值 | 说明 |
|--------|------|------|--------|------|
| 上下文窗口大小 | NInputNumber（每模型） | 1K–1M | 128,000 | 覆盖模型默认上下文窗口 |
| 最小保留轮数 | NSlider | 1–10 | 3 | 截断时最少保留的对话轮数 |
| CJK 系数 | NInputNumber（每模型） | 0.5–3.0 | 1.3 | 中日韩字符→token 换算系数 |
| 非CJK 系数 | NInputNumber（每模型） | 0.1–1.0 | 0.25 | 拉丁/符号→token 换算系数 |
| 结构化系数 | NInputNumber（每模型） | 0.2–1.5 | 0.5 | JSON/代码→token 换算系数 |

所有配置**即时保存**到 localStorage（`xuflow-config`），并实时 push 到 Rust 后端（`set_context_window` / `set_min_user_turns` Tauri 命令）。

## 九、待验证项

实际运行前建议验证以下场景：

1. **估算准确性**：发送中英混合 + 代码块消息，对比控制台中 `TokenUsage.before.estimated` 与 `TokenUsage.after.actual`，偏差应 <30%
2. **圆环动画**：连续对话，观察圆环从 12 点位置顺时针填充，颜色由绿→黄→橙→红过渡
3. **动态截断**：设置 contextWindow=2000，发送长消息，验证截断触发 + UI 标签闪现 + 百分比回落至 60% 以下
4. **轮次保护**：min_user_turns=2，发送 5 条 user 消息，验证保留最近 2 个完整轮次
5. **版本兼容**：前端优雅处理旧版空 payload（`agent:done`）不崩溃
6. **持久化**：修改上下文窗口/系数后重启应用，验证保留
